# exp_003_ema_slow

## 假設 (執行前手寫)

exp_002 增益全來自風險下降, CAGR 卡在 4.4%, 瓶頸在報酬端. 承接 exp_002 (adx=25), 只改一個變數:
slow_span 26 → 50. 慢線更長 → EMA 交叉更遲鈍 → 進出場都變慢 → 趨勢單較不被短期回彈洗出場,
騎更長趨勢段以提高 CAGR, 把 Sharpe 推向 1.0. slow=50 取業界慣例中期趨勢窗口 (先驗值, 非掃描挑最佳).
已預先識別風險: 進場更晚 (錯過趨勢初段)、出場更晚 (回吐更多頂部), 對 CAGR 淨效果方向不定.

## 實驗設置

- 數據: btc_usdt_1d.csv (與 exp_001/002 同一份)
- 策略: trend_following (承接 exp_002: fast 12, adx 25; 唯一改動 slow_span 26 → 50)
- 參數: {'fast_span': 12, 'slow_span': 50, 'adx_period': 14, 'adx_threshold': 25.0}
- 樣本內比例: 0.7 (與 exp_001/002 同一切法, 乾淨單變數對照)

## 結論 (執行後手寫)

**假設被推翻 (失敗實驗).** 樣本外對照 exp_002: Sharpe 0.97 → 0.89 (降), CAGR 4.4% → 3.9% (**假設要它升, 它反而降**),
最大回撤 -5.2% → -6.3% (變差), 盈虧比 3.72 → 3.00 (降). 唯一上升的是勝率 (41.7% → 50%), 但那反而是壞消息的線索
(見下). 延長慢線**沒有**解決報酬瓶頸, 還讓成績退步. exp_002 仍是 best-so-far.

**為什麼失敗 — 這才是這個實驗的價值**: 我預期的機制有兩股相反的力: (a) 持有更久 = 騎更長趨勢 (利 CAGR);
(b) 進場更晚 + 出場更晚 = 錯過趨勢初段 + 回吐更多頂部 (損 CAGR). 數據顯示 **(b) 壓過 (a)**. 關鍵證據:
交易筆數樣本外不變 (12 → 12), 表示不是「濾掉更多交易」, 而是**同樣那些趨勢進得更晚、退得更晚, 淨結果更差**.
勝率上升但 CAGR 與盈虧比下降, 正說明「贏的次數變多, 但每次贏得更少 / 吐回更多」— 晚進場砍掉了趨勢最肥的初段.

**這推翻了一個隱含假設**: 我們原以為「被短期回彈洗出好趨勢」是報酬的主要拖累. 數據說**不是** —
**進場時機 (吃到趨勢初段) 比避免洗盤更重要**. 用「放慢慢線」去騎更久, 代價是同時放慢了進場, 得不償失.

**這反而是 Option D (引擎移動止損) 的證據**: 若要「讓贏家跑更久」又不犧牲進場時機, 必須**把進場與出場解耦** —
快速進場 (維持 fast 12) + 只在逆走時才出場 (移動止損), 而非用單一慢線同時拖慢兩端. config-only 的旋鈕
把進出場綁死在一起, 這正是它失敗的根因. 要突破報酬瓶頸, 大概得動引擎那一步了.

局限: 同前, 只測 BTC 一資產一切法; 樣本外僅 12 筆交易, 樣本小, 0.89 vs 0.97 的差距也帶雜訊, 但方向 (變差) 清楚.
