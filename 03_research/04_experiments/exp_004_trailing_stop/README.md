# exp_004 移動止損 — 完整解讀 (第二次失敗, 但拼出了一個模式)

> 這份文件寫給完全沒接觸過量化交易 (quantitative trading, 用程式與數據做交易決策) 的人.

## 一句話總結

exp_003 失敗後我們推論: 想「抱住趨勢賺更多」, 正確做法是**讓進場保持快、只把出場放慢** (兩者用不同機關控制). 這一版就照這個思路, 給引擎加了一個真正的**移動止損 (trailing stop)** 當出場. 結果: **又失敗了, 而且比 exp_003 更慘.** 但這次失敗和上次**長得一模一樣** (勝率上升、賺的錢卻下降), 於是我們終於看清一件事: **在比特幣日線上, 「抱久一點」這條路本身就是死的 — 不管用什麼方法抱久, 都只會賺更少.**

這一版和前三個不同. 前三個只是改 config 的數字, 這一版**先給引擎蓋了一個新功能**, 再用它做實驗. 這兩件事的成敗要分開看:

| | 是什麼 | 結果 |
| --- | --- | --- |
| **工程 (做功能)** | 給回測引擎加「移動止損出場」機制 | ✅ **成功** — 功能正確, 通過完整自動化測試 |
| **實驗 (驗假設)** | 用這個功能能不能提高比特幣的報酬 | ❌ **失敗** — 假設不成立, 成績反而更差 |

**功能是好的, 只是「它能幫比特幣賺更多」這個假設錯了.** 這個移動止損功能未來在別的市場可能有用, 它不會因為這次實驗失敗就被丟掉. (工程細節見文末. )

## 什麼是移動止損 (trailing stop)?

回顧一下 exp_003 學到的教訓: 用「慢線長度」一個旋鈕去控制買賣, 會把**上車速度和下車速度綁死** — 想下車慢, 上車就跟著慢, 結果錯過趨勢初段, 得不償失.

**移動止損**就是專門用來**只放慢下車、不影響上車**的工具:

- **上車 (進場)**: 完全不變 — 還是用 exp_002 那套快速的 EMA + ADX 進場.
- **下車 (出場)**: 不再看均線交叉, 改成一條**會跟著獲利往上爬的地板價**:
  - 進場後, 記住價格到過的**最高點**.
  - 在最高點下方**固定距離** (這裡設 3 倍 ATR, 也就是 3 倍的「平常一天波動幅度」) 畫一條線.
  - 價格漲, 這條線就跟著往上抬 (但**永不下降**, 像棘輪).
  - 一旦價格跌破這條線, 才賣出.

> 生活類比: 你抓著一個往上飄的氣球, 手裡放一段固定長度的線. 氣球一直往上, 你就跟著往上; 只有當氣球**往下掉超過那段線的長度**, 你才鬆手. 這樣你能跟著趨勢一路往上, 又不會在頂部太貪心.

理論上這正是「讓贏家跑」的標準工具. 進場快、出場只在真正反轉時才觸發 — 聽起來完美.

## 結果: 又失敗了, 而且更慘

直接跟 exp_002 並排看**樣本外 (考卷)**:

| 指標 | exp_002 (EMA 出場) | exp_004 (移動止損) | 變化 | 怎麼解讀 |
| --- | --- | --- | --- | --- |
| **夏普比率 (Sharpe)** | 0.97 | **0.63** | ▼▼ 大跌 | 離 1.0 目標更遠了, 退步嚴重 |
| **年化報酬 (CAGR)** | 4.4% | **3.0%** | ▼ 變少 | **又是想讓它變多, 它卻變少** |
| **最大回撤 (Max Drawdown)** | -5.2% | **-6.2%** | ▼ 更深 | 帳戶縮水更嚴重 |
| **盈虧比 (Profit Factor)** | 3.72 | **2.99** | ▼ 變差 | 每賠 1 元賺回的變少 |
| **勝率 (Win Rate)** | 41.7% | **55.6%** | ▲ 上升 | 唯一變好的 — 又是它 |
| **交易筆數** | 12 | **9** | ▼ 更少 | 抱更久, 進出更少 |

**exp_002 依然是最好的版本.** 這一版不但沒進步, 還明顯退步.

## 破案: 兩次失敗長得一模一樣 → 這是一個「模式」

把 exp_003 和 exp_004 擺在一起, 你會發現它們的失敗**形狀完全相同**:

| | exp_003 (放慢慢線) | exp_004 (移動止損) |
| --- | --- | --- |
| 想達成 | 抱久一點 → 賺更多 | 抱久一點 → 賺更多 |
| 勝率 | ▲ 上升 | ▲ 上升 |
| 年化報酬 | ▼ **下降** | ▼ **下降** |
| 結論 | 失敗 | 失敗 |

**兩個完全不同的機制, 用完全不同的方法去「抱久一點」, 卻失敗得一模一樣.** 這不是巧合. 當兩條獨立的路都撞上同一面牆, 那面牆就是真的:

> **在比特幣日線上, 「讓贏家跑更久」這個想法本身就是錯的. 不管你用什麼方法抱得更久, 結果都是「贏的次數變多、但每次賺更少」, 總報酬反而下降.**

還有一個關鍵證據讓我們更確定: 這一版連**練習卷 (樣本內) 都變差了** (夏普從 exp_002 的 1.54 掉到 1.14). 如果只有考卷變差, 我們還能懷疑是運氣; 但**練習卷和考卷一起變差**, 代表這是移動止損的**真實效果**, 不是抽樣運氣.

### 為什麼「抱久一點」在比特幣行不通?

移動止損要等價格從高點**回吐 3 倍 ATR** 才賣. 但比特幣的日線趨勢一旦結束, 往往是**急轉直下、快速反轉**. 對這種行情:
- **均線交叉出場 (exp_002)** 是對「趨勢轉弱」本身反應 → 常常出得**更早、更好**.
- **移動止損 (exp_004)** 硬要等固定的回吐幅度 → 每次觸頂都先白白吐掉一段, 還可能抱著它一路跌到止損才走.

換句話說: **比特幣趨勢的「頂」很尖, 反應轉弱訊號的均線, 打不過死等固定回吐的移動止損.**

## 最大的收穫: 我們終於找到瓶頸不在哪

exp_002、003、004 加起來, 排除了一整個方向:

- exp_002: 加過濾 (ADX) → 有用, 但只改善風險, 不改善報酬. **這是目前最佳.**
- exp_003: 放慢進出場想抱久 → **失敗.**
- exp_004: 移動止損想抱久 → **失敗.**

**三個實驗證明: 報酬瓶頸不在「出場」, 也不在「抱多久」.** exp_002 的均線交叉出場, 在我們試過的方法裡已經接近最好. 繼續在「怎麼賣、抱多久」上打轉, 已經被數據判了死刑.

合理的推論是: **這套「EMA + ADX 趨勢跟蹤」在比特幣日線上的簡單形態, 夏普天花板大概就在 1.0 附近, 而 exp_002 (0.97) 已經頂到那裡了.** 想真正突破 1.0, 大概得**換一種賺錢的邏輯 (edge)**, 而不是繼續微調同一套 (見「下一步」).

## 應該注意什麼 (行外人最容易誤會的地方)

- ⚠️ **失敗 ×2 = 一個更強的結論.** 單次失敗可能是運氣或參數沒調好. 但兩個獨立方法**同型失敗**, 是在告訴你一個結構性的真相. 這種「多路撞同一面牆」的證據, 比單次成功可靠得多.
- ⚠️ **「功能做出來」不等於「假設成立」.** 這一版的移動止損功能是對的、能用的; 錯的只是「它能幫比特幣賺更多」的猜測. 別把工程的成功和實驗的成功混為一談.
- ⚠️ **又一次: 勝率高 ≠ 賺得多.** 這已經是連續第二個實驗用血淚證明這件事. 只盯勝率會被反覆騙到.
- ⚠️ **知道「此路不通」極有價值.** 這三個實驗最大的產出, 不是找到好策略, 而是**確定了報酬瓶頸不在出場端** — 這讓未來不必再浪費時間在這個方向. 排除錯路, 就是進展.
- ⚠️ **樣本仍小.** 樣本外只有 9 筆交易, 移動止損也只試了 3 倍 ATR 一個寬度. 但因為「樣本內外同向變差」加上「與 exp_003 同模式」, 我們有把握說這個方向沒用, 而不只是這一個參數沒調好.

## 相關概念小詞典 (只列這一版重點; 其餘見前三個實驗)

| 名詞 | 白話解釋 |
| --- | --- |
| 移動止損 (trailing stop) | 一條跟著獲利往上爬、永不下降的地板價; 價格跌破才賣, 用來讓趨勢單抱更久 |
| ATR 平均真實波幅 | 「平常一天大概波動多少」; 3 倍 ATR = 給價格 3 天份的正常波動當緩衝 |
| 棘輪 (ratchet) | 只能往一個方向走的機制; 這裡指止損線只升不降 |
| 模式 (pattern) | 多次獨立實驗出現相同結果; 比單次結果可靠得多 |
| edge (優勢/賺錢邏輯) | 策略之所以能長期賺錢的根本原因; 微調參數換不來新的 edge |
| 天花板 (ceiling) | 一套方法在某市場能達到的績效上限, 再怎麼調也過不去 |

## 這次動的不只 config — 還有引擎 (工程細節)

前三個實驗只改 config 的數字, 這一版**第一次動到共用的回測引擎**, 加了移動止損出場功能. 因為引擎是所有實驗共用的地基, 動它必須格外小心, 所以我們用了嚴謹的做法:

- **測試先行 (TDD, Test-Driven Development)**: 先寫 9 條自動化測試 (描述移動止損「應該」怎麼動), 確認它們**失敗**, 再寫功能讓它們**通過**. 這樣才能確定測試真的在檢驗東西, 而不是事後補的橡皮圖章.
- **測了什麼**: 大跌會觸發出場、上漲不會誤觸發、出場後不會立刻買回、真反轉前會一路抱著、以及最重要的 —— **不會偷看未來數據** (移動止損這種「看走勢決定」的邏輯最容易犯這個錯).
- **向後相容**: 這個功能**預設關閉**. 關閉時, 引擎行為和加功能前**逐位元完全一致** — 我們重跑了 exp_001/002/003, 數字一個都沒變, 有測試鎖死這點.
- **結果**: 全套 43 條測試通過. 功能是可靠的.

config 上的唯一改動是引擎參數多了一行:

```python
ENGINE_PARAMS = {
    # ... 其餘與 exp_002 相同 ...
    "trailing_stop_atr_multiplier": 3.0,   # ← 啟用移動止損出場; exp_002 沒有這行 (預設關閉)
}
```
