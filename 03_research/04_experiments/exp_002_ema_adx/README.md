---

## 一句話總結

exp_001 那套「追漲殺跌」的規則, 有個毛病: **市場橫著盤整、沒方向的時候, 它還是照樣進進出出, 白白被來回巴掌 (假信號) 打到虧錢**. 這一版加了一個**趨勢強度過濾器**, 規定「只有當市場真的走出明確方向時才准進場」. 結果: **帳戶更平穩了 (回撤更小、勝率更高), 但每年賺的錢幾乎沒變多** — 進步幾乎全來自「少賠」, 不是「多賺」. 樣本外夏普從 0.80 升到 **0.97, 逼近但仍差 1.0 一步**.

---

## 這個實驗想回答的問題

exp_001 的結論是「有戲, 但不夠好」: 樣本外 (out-of-sample, 沒見過的數據) 夏普只有 0.80, 沒達到專案設定的 1.0 門檻. 我們回頭看它的交易, 提出一個懷疑:

> 「會不會很多虧損, 是在**市場沒方向、橫盤震盪**的時候發生的? 那種時候快慢均線會不停假交叉, 一下叫你買一下叫你賣, 兩邊挨耳光. 」

如果這個懷疑成立, 那解法很直接: **加一個開關, 只在趨勢夠強的時候才准做, 盤整的時候一律空手在旁邊看.** 這個「趨勢夠不夠強」的量尺, 就是 **ADX**.

這一版就是要驗證: **加上這個過濾, 能不能把樣本外夏普推過 1.0?**

## 這一版到底改了什麼 (只改一個旋鈕)

這是本專案的鐵律: **一次只改一個變數**, 這樣成績變好或變壞, 才能確定是「這個改動」造成的, 而不是別的東西混進來.

| | exp_001 (基準版) | exp_002 (這一版) |
| --- | --- | --- |
| 數據 | BTC/USDT 日線 | **一模一樣** |
| 進出場邏輯 | EMA 快 12 / 慢 26 交叉 | **一模一樣** |
| 買多少、止損、成本 | 每筆賭 1%, 2×ATR 止損, 扣費 | **一模一樣** |
| 練習卷/考卷切法 | 前 70% / 後 30% | **一模一樣** |
| **趨勢過濾** | **無 (ADX 門檻 = 0)** | **有 (ADX 門檻 = 25)** ← 唯一改動 |

換句話說, 這兩個實驗之間**只有一個字不同**: 進場前多問一句「現在趨勢夠強嗎?」

---

## ADX 是什麼? (這一版的主角)

**ADX (Average Directional Index, 平均趨向指標)** 是一把「趨勢強度的尺」. 它只回答一個問題: **現在這個市場, 是有明確方向 (不管漲還是跌), 還是在原地橫盤打轉?**

- ADX **數值高** → 趨勢明確、走得有勁 (可能是大漲, 也可能是大跌, 它不管方向, 只管「有沒有勁」)
- ADX **數值低** → 沒方向、在盤整、上下亂晃

業界有個約定俗成的分界: **ADX > 25 就算「趨勢明確」**. 我們就直接採用這個慣例值當門檻.

> 生活類比: 均線交叉像是「有人喊了一聲『往這走!』」, 而 ADX 像是「先確認這聲喊叫是不是來自一個真的知道路、走得很堅定的人」. 如果喊的人自己都在原地打轉 (ADX 低), 那就別跟.

**新規則變成兩個條件都要成立才進場:**
1. 快線在慢線之上 (方向偏多) — 這是 exp_001 就有的
2. **而且** ADX > 25 (趨勢夠強) — 這是這一版新加的

只要盤整 (ADX ≤ 25), 就算均線喊買, 我們也**按兵不動**.

## 結果: 有沒有變好?

先看最重要的**樣本外 (考卷, 只有這個才算數)**, 直接跟 exp_001 並排:

| 指標 | exp_001 (無過濾) | exp_002 (加 ADX 過濾) | 變化 | 怎麼解讀 |
| --- | --- | --- | --- | --- |
| **夏普比率 (Sharpe)** | 0.80 | **0.97** | ▲ +0.17 | 風險調整後報酬. 明顯進步, 但仍未過 1.0 |
| **年化報酬 (CAGR)** | 4.5% | **4.4%** | ≈ 幾乎不變 | 每年賺的錢**沒有變多** ← 關鍵 |
| **最大回撤 (Max Drawdown)** | -6.7% | **-5.2%** | ▲ 更淺 | 帳戶最慘縮水變小, 更平穩 |
| **勝率 (Win Rate)** | 37.5% | **41.7%** | ▲ 上升 | 賺錢的交易比例提高 |
| **盈虧比 (Profit Factor)** | 2.46 | **3.72** | ▲ 明顯上升 | 每賠 1 元能賺回的錢變多 |
| **交易筆數** | 16 | **12** | ▼ 減少 4 筆 | 濾掉了 4 次進場, 符合「少做假信號」的預期 |

**六項指標裡, 五項往好的方向動, 一項 (CAGR) 原地踏步.** 方向完全符合假設: 少做了盤整時的無謂交易, 該賠的少賠了, 剩下的交易品質更高.

---

## 最重要的一句誠實話: 進步來自「少賠」, 不是「多賺」

這是整個實驗最該記住的一點, 也是行外人最容易看錯的地方.

夏普從 0.80 升到 0.97, 看起來策略「變強了」. 但請看**年化報酬幾乎沒變 (4.5% → 4.4%)**. 這說明:

> **我們並沒有因為這個過濾器而賺到更多錢. 夏普之所以上升, 完全是因為「分母」變小了 — 也就是波動和回撤變小、帳戶走得更平穩.**

打個比方: 兩個人年底都賺了差不多的錢, 但一個人過程中大起大落、心臟受不了 (exp_001), 另一個人穩穩當當、睡得著覺 (exp_002). 夏普獎勵的是後者. **ADX 過濾器買到的是「安穩」, 不是「更肥的獲利」.**

這件事很重要, 因為它直接告訴我們**下一步該往哪使力**: 如果想再把成績往上推, 光靠「濾掉更多爛交易」大概到頂了 (爛交易就那麼多), **瓶頸在於怎麼從好交易裡榨出更多報酬** — 那是止損、出場邏輯的事, 不是過濾器的事. (見文末「下一步」. )

## 那到底算不算成功?

**算部分成功, 但沒有通關.**

1. ✅ **假設的機制被證實**: 「盤整假交叉在拖後腿」這個懷疑是對的. 濾掉它們, 帳戶確實更穩、更少受傷.
2. ✅ **比基準明確更好**: 五項指標改善, 沒有一項變差. 這一版全面優於 exp_001.
3. ❌ **仍未達標**: Phase 2 的門檻是樣本外夏普 **> 1.0**, 我們拿到 **0.97** — 逼近, 但差臨門一腳.
4. ⚠️ **別過度解讀 0.97 這個數字**: 樣本外只有 **12 筆交易**, 樣本很小. 0.97 和 1.0 的差距, 老實說落在**雜訊 (運氣) 範圍**內. 我們可以說「這一版比較好」, 但不能拍胸脯說「它剛好差一點點就通關」— 換一段行情, 這個數字可能上下跳動不小.

一個額外的好消息: 這一版**練習卷 (樣本內) 到考卷 (樣本外) 的成績衰減比較溫和** (夏普 1.54→0.97, 保住 63%; exp_001 是 1.45→0.80, 只保住 55%), 而且練習卷的最大回撤大幅收斂 (-10.3% → -5.0%). 衰減越小, 通常代表策略越穩健、越不像是「背答案背出來的」(過擬合).

---

## 應該注意什麼 (行外人最容易誤會的地方)

- ⚠️ **「夏普變高」不等於「賺更多」.** 這一版就是活生生的例子: 錢賺得一樣多, 只是波動變小, 夏普就上去了. 看策略永遠要同時看報酬和風險兩面.
- ⚠️ **0.97 沒有「差一點就 1.0」那麼激勵人心.** 在 12 筆交易的小樣本下, 這個排名帶著運氣成分. 不要因為「好接近喔」就腦補它其實已經夠好.
- ⚠️ **只測了 BTC 一個資產、一段行情.** ADX 過濾在 BTC 這段數據有用, 不代表在以太幣 (ETH) 或美股 (SPY, QQQ) 也有用. 下一步一定要跨市場重測, 才知道這是真本事還是巧合.
- ⚠️ **加過濾器 = 交易變少.** 從 16 筆掉到 12 筆. 交易越少, 每一筆的運氣佔比越重, 統計上越不可靠. 這是所有「加條件過濾」策略的通病 — 濾得越乾淨, 樣本越小.

## 相關概念小詞典 (只列這一版新增的; 其餘見 exp_001)

| 名詞 | 白話解釋 |
| --- | --- |
| ADX 平均趨向指標 | 一把量「趨勢有多強」的尺, 只管有沒有勁, 不管漲跌方向. > 25 慣例上算趨勢明確 |
| 過濾器 (filter) | 在原本的進場規則上多加一個條件, 把不符合的機會擋掉、不做 |
| 盤整 / 橫盤 (choppy market) | 市場沒明確方向, 上下亂晃, 追漲殺跌策略最容易在這裡挨巴掌 |
| 假信號 (false signal) | 均線交叉叫你進場, 但趨勢沒真的來, 很快反向, 害你小虧出場 |
| 風險調整後報酬 | 不只看賺多少, 還看「為了賺這些, 承擔了多少波動」. 夏普就是一種 |

## config.py 改了哪一行

整個 `config.py` 與 exp_001 幾乎一字不差, **唯一的改動**是策略參數裡的一行:

```python
STRATEGY_PARAMS = {
    "fast_span": 12,
    "slow_span": 26,
    "adx_period": 14,
    "adx_threshold": 25.0,   # ← exp_001 是 0.0 (不過濾); 這裡改成 25 (只在趨勢明確時進場)
}
```

`adx_threshold` 的含義: ADX 要**高過這個值**才准進場. 設 `0` 等於這個條件永遠成立 = 不過濾 (就是 exp_001). 設 `25` 就啟用過濾. 調更高 (如 30, 35) 會更嚴格、交易更少、更保守; 調更低則接近不過濾. 其餘每個參數的意義, 詳見 exp_001 README 的「config.py 參數詳解」.

## 下一步

這一版的結論指出了一條清楚的線索: **增益來自「少賠」, 報酬端 (每年賺多少) 卡住沒動.** 所以要再往上突破 1.0, 該去動的是**怎麼讓賺錢的趨勢單跑得更久、賺得更多**, 而不是繼續加過濾器.

- **exp_003**: 把目前「固定 2×ATR 就止損出場」改成**移動止損 (trailing stop, 隨獲利往上抬的止損)** 或放寬止損倍數, 讓抓對的大趨勢別太早被洗出場, 試著把年化報酬 (CAGR) 推上去、夏普過 1.0. 一樣只改這一個變數.
- **跨市場驗證**: 把 exp_001 / exp_002 同一組參數搬到 ETH/USDT、SPY、QQQ 上重跑 (Round 1 其餘基準), 看 ADX 過濾的好處是「跨市場都成立的真本事」, 還是「只在 BTC 這段行情剛好有用的巧合」.
