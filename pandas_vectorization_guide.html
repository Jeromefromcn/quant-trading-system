Pandas 向量化數據操作

從 Java / Go 的循環思維，切換到 pandas 的向量化思維

預計 1–2 週 Python 3.11+ pandas · numpy 量化前置技能
!

思維方式切換：最重要的一步

這不是語法問題，是解決問題方式的根本切換

你在 Java / Go 裡的習慣是「逐行遍歷數據，對每一行做操作」。pandas 的向量化思維是「把整列數據視為一個對象，一次性描述對整列的操作」。這個切換才是學習 pandas 最難的地方。

⚠ 核心認知：向量化 ≠ 語法糖
為什麼
向量化操作由底層 C/Fortran 執行，速度比 Python for loop 快 100–1000 倍。在量化場景裡，你會處理幾十萬行 OHLCV 數據，速度差距直接影響策略回測的可行性。
怎麼用
每當你想寫 for i in range(len(df))，先停一下——問自己：「這個操作能不能用一個 pandas 方法描述？」90% 的情況答案是可以。

思維對比

❌ 舊思維（Java / Go 習慣）
# 逐行計算收益率
returns = []
for i in range(1, len(df)):
    r = df['close'][i] / df['close'][i-1] - 1
    returns.append(r)
df['return'] = [None] + returns

Python for loop，100k 行約需 200ms+

✓ 向量化思維（pandas 方式）
# 整列一次操作
df['return'] = df['close'].pct_change()

底層 C 執行，同樣數據約需 1–2ms

💡 類比：如果說 Go 的 for loop 是「親自去倉庫逐一搬貨」，那向量化就是「告訴叉車把整托盤貨一次搬走」。你描述的是意圖，底層決定怎麼執行。

核心操作清單

量化工作 95% 的數據操作，都由這些方法組合完成

數據結構基礎

概念類比（Go）說明
Series []float64 + 索引 一維帶標籤數組，類似一列數據
DataFrame []map[string]float64 二維表格，每列是一個 Series；量化中的 OHLCV 就是一個 DataFrame
df.index map 的 key 通常是時間戳（DatetimeIndex），用於對齊不同數據源
df.dtypes struct 的欄位型別 每列的數據類型，float64 / int64 / object 等

位移與滾動（量化最常用）

方法語義典型用途
.shift(n) 把整列數據向下位移 n 行（正數向下，負數向上） 計算昨日收盤價、lag 特徵、收益率
.rolling(n) 以 n 為窗口大小，對每個位置取前 n 行做聚合 移動平均、滾動標準差、滾動最大值
.expanding() 從第一行到當前行的累積窗口 累積最大回撤、累積收益
.ewm(span=n) 指數加權移動窗口 EMA（指數移動平均）
.diff(n) 當前值 - 前 n 行的值 價格變動量、動量計算

篩選與布林索引

# 篩選收盤價 > 50000 的行
high_price = df[df['close'] > 50000]

# 多條件篩選（注意用 & | 而不是 and or）
signal_rows = df[(df['ma5'] > df['ma20']) & (df['volume'] > df['volume'].mean())]

# .loc 用標籤，.iloc 用位置索引
df.loc['2024-01-01':'2024-06-30', 'close']   # 按日期範圍取一列
df.iloc[-20:, :]                              # 最後 20 行

分組、合併、重採樣

方法說明
.groupby('col').agg(...) 按某列分組再聚合，例如按月計算平均收益率
pd.concat([df1, df2]) 縱向拼接兩個 DataFrame（行數增加）
df1.merge(df2, on='date') 按日期欄位做 JOIN，合併兩個數據源
.resample('W').agg(...) 時間序列重採樣，日線轉周線

Apply（向量化不夠時的退路）

⚠ .apply() 本質上仍是 Python 循環，比原生向量化慢很多。只在邏輯無法向量化時使用，不要把它當預設選項。 
# 只在邏輯複雜、無法向量化時才用 apply
def classify_candle(row):
    if row['close'] > row['open']:
        return 'bullish'
    return 'bearish'

df['candle_type'] = df.apply(classify_candle, axis=1)

# 上面這個其實可以向量化：
df['candle_type'] = (df['close'] > df['open']).map({True: 'bullish', False: 'bearish'})
02

量化場景示例

把核心操作應用到實際策略計算中

收益率與風險指標

import pandas as pd
import numpy as np

# 假設 df 是 OHLCV DataFrame，index 為 DatetimeIndex

# 日收益率
df['daily_return'] = df['close'].pct_change()

# 累積收益率
df['cum_return'] = (1 + df['daily_return']).cumprod() - 1

# 年化夏普比率（假設無風險利率為 0）
sharpe = df['daily_return'].mean() / df['daily_return'].std() * np.sqrt(252)

# 最大回撤
rolling_max = df['close'].expanding().max()
drawdown    = df['close'] / rolling_max - 1
max_dd      = drawdown.min()

技術指標計算

# 移動平均
df['ma5']  = df['close'].rolling(5).mean()
df['ma20'] = df['close'].rolling(20).mean()
df['ema12'] = df['close'].ewm(span=12, adjust=False).mean()

# 布林帶
df['bb_mid']   = df['close'].rolling(20).mean()
df['bb_std']   = df['close'].rolling(20).std()
df['bb_upper'] = df['bb_mid'] + 2 * df['bb_std']
df['bb_lower'] = df['bb_mid'] - 2 * df['bb_std']

# RSI（14 日）
delta = df['close'].diff()
gain  = delta.clip(lower=0).rolling(14).mean()
loss  = (-delta.clip(upper=0)).rolling(14).mean()
df['rsi'] = 100 - (100 / (1 + gain / loss))

信號生成

# MA 金叉死叉信號（不用 if-else，直接比較兩列）
df['signal'] = 0
df.loc[df['ma5'] > df['ma20'], 'signal'] = 1   # 做多
df.loc[df['ma5'] < df['ma20'], 'signal'] = -1  # 做空

# 只在信號發生變化時觸發（去除持倉期間的重複信號）
df['trade_signal'] = df['signal'].diff().fillna(0)
✓ 關鍵習慣：先在小數據集（100-200 行）上驗證邏輯，確認結果正確後再跑全量數據。用 df.head(20) 和 print 是你最好的調試工具。

常用模式速查

遇到問題先查這裡，不要先搜 Stack Overflow
需求寫法
昨日收盤價 df['close'].shift(1)
N 日最高價 df['high'].rolling(N).max()
欄位是否大於前一日 df['close'] > df['close'].shift(1)
刪除含 NaN 的行 df.dropna()
用前值填充 NaN df.fillna(method='ffill')
按日期排序 df.sort_index()
取某段時間 df.loc['2024-01':'2024-06']
日線轉周線（取最後一日收盤） df['close'].resample('W').last()
計算列的百分位排名 df['volume'].rank(pct=True)
兩列元素級比較取較大值 df[['col1','col2']].max(axis=1)
條件賦值（類似三目運算） np.where(condition, val_true, val_false)
查看缺失值數量 df.isnull().sum()

常見陷阱

⚠
SettingWithCopyWarning — 對切片賦值會出現此警告，可能導致原 DataFrame 未被修改。
解決：用 .loc[condition, 'col'] = value 替代 df[condition]['col'] = value。
⚠
rolling 的前 N-1 行是 NaN — .rolling(20) 的前 19 行沒有足夠數據，結果為 NaN。
在回測時記得用 df.dropna() 或從第 20 行開始計算信號。
⚠
未來數據洩漏（Look-ahead bias） — .rolling() 預設是「往前看」，不會洩漏未來數據。但 shift(-1)（負數）會用到未來數據，在策略信號計算時要小心。

自我驗收標準

能用「不超過 10 行、零 for loop」完成這三道題，即達標

這三道題覆蓋了量化工作最常用的數據操作模式。完成後說明你已完成思維切換。

題 1：滾動最大回撤

給定一個 close 列，計算每個時間點「從歷史最高點到當前的最大回撤」，結果存入 df['drawdown']。

驗收條件
  • 不超過 3 行代碼
  • 零 for loop，零 if-else
  • 用 df['close'].expanding().max() 作為中間步驟

題 2：篩選極端收益日

找出所有「當天收益率排名在全部交易日前 10%」的交易日（即收益率最高的 10% 的日子），返回對應的 DataFrame 子集。

驗收條件
  • 不超過 2 行代碼
  • 用到布林索引和 .quantile() 或 .rank(pct=True)

題 3：合併不同頻率數據

你有兩個 DataFrame：df_daily（日線，OHLCV）和 df_weekly（周線，只有 close 和 volume）。把兩者合併到一個 DataFrame，日線數據每行都要包含「當週」的周線數據。

驗收條件
  • 不超過 5 行代碼
  • 使用 .resample() 或 .merge_asof()
  • 結果不包含未來數據（不能用未來的周線數據填充過去的日線）
