## [2026-06-10] 使用 Monorepo 結構

**決策: ** 學習到實盤全用一個倉庫, 不提前拆分.

**理由: **
- 學習 → 研究 → 實盤是自然演進關係, 拆分會製造同步問題
- Git history 就是完整研究記錄, 拆庫會斷掉連續性
- 獨立開發者, monorepo 的協作管理成本接近零

**什麼時候才拆: ** 引入協作者, 或有 CI/CD 安全合規要求(預計至少一年後)
