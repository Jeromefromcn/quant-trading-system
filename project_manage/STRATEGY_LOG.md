## 方法論註記 (methodology note) — 樣本外不是「只考一次的考卷」

嚴謹流程該切**三份**, 但目前 pipeline 只切**兩份** (樣本內 70% / 樣本外 30%), 沒有保留集. 因此紀律如下:

- **現在的「樣本外」= 驗證集 (validation set), 可重複用.** 它是設計來反覆比較候選策略的. exp_001, exp_002 都跑同一段 BTC 樣本外 — 合法, 但每重複一次, 這段數字就多帶一分**樂觀偏誤 (optimistic bias)**, 因為我的決策 (看到 exp_001 沒過關才加 ADX) 已洩漏進去. 它**不是**通關判準.
- **BTC 樣本外已被用 ≥ 2 次, 且無法再從 BTC 乾淨補出保留集** (尾段已在看過的 30% 內, 已污染).
- **真正未污染的「只考一次」資源只剩兩個, 用一次少一次, 不准拿去反覆調參:**
  1. **ETH / SPY / QQQ** — 至今零實驗, 乾淨. 當跨市場最終驗證用, 不當調參場.
  2. **Phase 3 紙上交易 (paper trading)** — 未來數據, 物理上偷看不了, 才是真期末考.
- **回測樣本外再漂亮, 都只是彩排.** 策略定稿、要上線前, 才動保留集 / 紙上交易那唯一一次.

### 核心原則: 目標不是「永遠看不到」, 而是「決策凍結前不看」

- **沒有任何歷史數據能保證永遠不看** — 擁有它就是為了遲早測它, 一測就花掉, 且造不出更多過去. 歷史數據是**一次性消耗品**.
- 保留集的價值不在「沒被看過」, 而在**它沒影響過任何決策**. 只要在策略參數/規則**全部凍死之後**才跑那唯一一次, 就是無偏估計. 可達成的保證是「凍結前不碰」, 不是「永不碰」.
- **跨市場驗證 = 消費那個市場.** 鐵律: 先把 BTC 策略完全定稿, 再跑 ETH; **看完 ETH 結果不准回頭改參**, 否則 ETH 從保留集淪為驗證集, 當場燒掉. 每個乾淨市場只跑一次, 結果照單全收.
- **唯一可再生的乾淨數據是未來.** 歷史有限 (花一份少一份, 靠自律防偷看, 自律必然鬆動); 未來無限 (每天長新數據, 時間之箭物理上禁止偷看). 別囤積歷史, Phase 3 向前實測才是取之不盡的期末考.
- **正確的花費順序**: 已污染的 BTC 上調參 → 定稿 → 乾淨市場各跑一次 → Phase 3 向前實測. 讓每份數據「唯一一次乾淨的看」發生在對的時機.

### 跨市場 config 注意事項 (股票 ≠ 加密)

代碼相容四個市場 (欄位一致, 直接跑), 但跑股票時 config.py **必須改對, 否則跑得出來卻算錯**:

- `trading_days_per_year`: 加密 **365**, 美股 (SPY/QQQ) **252**. 用錯會讓年化報酬與 Sharpe 全部算錯.
- `fee_rate` / `slippage_rate`: 目前是 Binance 費率 (0.1% / 0.05%). 美股券商費率不同 (常近零佣金), 沿用會高估股票成本.
- 數據層: SPY/QQQ 歷史較短 (約 1500 根 vs 加密 3245 根), 樣本外更小; 開頭偶有退化 K 線 (四價相等, ATR=0), 代碼有 clip 不崩但開頭幾根倉位失真, 影響輕微.

## [2026-07-06] 實驗 #002: 加 ADX > 25 趨勢過濾, 濾掉盤整假交叉能否把樣本外 Sharpe 推過 1.0

### 假設

exp_001 純 EMA 12/26 交叉樣本外 Sharpe 只有 0.80 (未過 1.0 門檻), 勝率僅 37.5%. 懷疑虧損多來自盤整市 (choppy market) 的假交叉. 加入 ADX (Average Directional Index, 平均趨向指標) 過濾, 只在 ADX > 25 (趨勢明確) 時進場, 應能減少假信號 → 交易變少、勝率與 Sharpe 上升. 相對 exp_001 只改 adx_threshold 一個變數.

### 實驗設置

- 資產: BTC/USDT
- 時間框架: 日線 (1d), 2017-08 至 2026-07
- 樣本內: 前 70% (調參用)
- 樣本外: 後 30% (只驗證)
- 關鍵參數: EMA 快 12 / 慢 26, **adx_threshold 0 → 25** (唯一改動), 其餘引擎參數同 exp_001

### 結果

| | 樣本內 | 樣本外 |
|----------|--------|--------|
| 年化收益 | 8.8% | 4.4% |
| 最大回撤 | -5.0% | -5.2% |
| Sharpe | 1.54 | 0.97 |
| 勝率 | 55.2% | 41.7% |
| 盈虧比 | 9.92 | 3.72 |
| 交易筆數 | 29 | 12 |

樣本外對照 exp_001: Sharpe 0.80→0.97, 回撤 -6.7%→-5.2%, 勝率 37.5%→41.7%, 盈虧比 2.46→3.72, 交易 16→12.

### 結論

假設**方向成立但未通關**. 六項指標五項改善, 「濾掉盤整假交叉」機制有效. 關鍵誠實觀察: 年化報酬幾乎不變 (4.5%→4.4%), Sharpe 上升**幾乎全來自風險下降** (回撤、震盪減少), 不是賺更多 — ADX 過濾買到的是「安穩」不是「多賺」. 樣本外 0.97 仍未過 1.0, 且在僅 12 筆交易的小樣本下, 0.97 與 1.0 差距落在雜訊內, 不宜過度解讀. 正面訊號: 樣本內→外衰減比 exp_001 溫和 (保住 63% vs 55%), 較不像過擬合. 局限: 只測 BTC 一資產一切法.

### 下一步實驗

#003: 增益卡在報酬端, 瓶頸不在過濾而在出場. 把固定 2×ATR 止損換成移動止損 (trailing stop) 或放寬止損倍數, 讓趨勢單跑更久, 試著把 CAGR 與 Sharpe 推過 1.0. 另需把 exp_001/002 參數跑 ETH/USDT, SPY, QQQ 檢驗跨市場穩定性.

> 註: 開 exp_003 前發現引擎**無止損出場** — atr_stop_multiplier 只用於算倉位大小, 出場只靠 EMA 交叉翻空. 故「換移動止損」不是 config 改動, 而是引擎新功能. exp_003 改走 config-only 路線: 延長慢線讓趨勢單抱更久.

## [2026-07-06] 實驗 #003: 延長慢線 (slow_span 26→50) 讓趨勢單抱更久, 能否提高 CAGR 突破報酬瓶頸

### 假設

exp_002 增益全來自風險下降, CAGR 卡在 4.4%, 瓶頸在報酬端. 承接 exp_002 (adx=25), 只改 slow_span 26→50. 慢線更長 → 進出場更遲鈍 → 趨勢單較不被短期回彈洗出場 → 騎更長趨勢段提高 CAGR. slow=50 取業界慣例中期趨勢窗口 (先驗值, 非掃描). 已預先識別風險: 進場也會變晚, 對 CAGR 淨效果方向不定.

### 實驗設置

- 資產: BTC/USDT
- 時間框架: 日線 (1d), 2017-08 至 2026-07
- 樣本內: 前 70%
- 樣本外: 後 30%
- 關鍵參數: fast 12, **slow 26 → 50** (唯一改動), adx 25 (承接 exp_002), 其餘同 exp_001/002

### 結果

| | 樣本內 | 樣本外 |
|----------|--------|--------|
| 年化收益 | 8.7% | 3.9% |
| 最大回撤 | -5.0% | -6.3% |
| Sharpe | 1.53 | 0.89 |
| 勝率 | 57.1% | 50.0% |
| 盈虧比 | 8.16 | 3.00 |
| 交易筆數 | 28 | 12 |

樣本外對照 exp_002: Sharpe 0.97→0.89, CAGR 4.4%→3.9%, 回撤 -5.2%→-6.3%, 盈虧比 3.72→3.00, 交易 12→12 (持平).

### 結論

**假設被推翻 (失敗實驗).** 延長慢線沒提高 CAGR, 反而讓 CAGR 與 Sharpe 雙降, exp_002 仍是 best-so-far. 破案關鍵: 勝率上升 (41.7%→50%) 但 CAGR/盈虧比下降 + 交易筆數不變 (12→12) → 進的是**同樣那些趨勢, 只是上車更晚**, 錯過趨勢最肥的初段. 這**推翻了一個隱含假設**: 「被回彈洗出場 (下車太早)」不是報酬主因, **「進場太晚」才更貴**. 用慢線長度這一個旋鈕會把進場與出場速度綁死 — 想下車慢, 上車就跟著慢, 得不償失. 局限: 只測 BTC, 樣本外僅 12 筆.

### 下一步實驗

#004 候選: 這次失敗反而是**引擎移動止損 (trailing stop) 的證據** — 要「讓贏家跑」又不犧牲進場時機, 必須**解耦進出場** (維持 fast 12 快進場, 出場改移動止損). 這是引擎功能非 config 實驗, 需寫測試, 屬 build ahead, 要另評估是否此刻做. 若暫不動引擎, config-only 已無明顯槓桿可攻報酬端.

## [2026-07-06] 實驗 #004: 移動止損 (3×ATR) 取代 EMA 出場, 解耦進出場能否讓贏家跑、突破報酬瓶頸

### 假設

exp_003 證明放慢慢線會連進場一起拖慢. 承接 exp_002 (best-so-far), 進場完全不動, 唯一改動: 出場從 EMA 下穿換成 3×ATR 移動止損 (引擎新功能, 取代 EMA 出場). 預期快進場 + 只在真反轉才出 → 抱住趨勢更久拉高 CAGR, Sharpe 過 1.0. 已預先識別風險: 移動止損頂部回吐、盤整可能更差.

### 實驗設置

- 資產: BTC/USDT, 日線, 2017-08 至 2026-07
- 樣本內/外: 70% / 30%
- 關鍵參數: 策略同 exp_002 (fast 12, slow 26, adx 25); 引擎 **+trailing_stop_atr_multiplier=3.0** (唯一改動)
- 工程: 引擎移動止損為新功能, 以 TDD 開發 (9 條行為+前視截斷+向後相容測試先行, 全套 43 passed, exp_001/002/003 重跑逐位元不變)

### 結果

| | 樣本內 | 樣本外 |
|----------|--------|--------|
| 年化收益 | 6.1% | 3.0% |
| 最大回撤 | -5.9% | -6.2% |
| Sharpe | 1.14 | 0.63 |
| 勝率 | 47.8% | 55.6% |
| 盈虧比 | 4.28 | 2.99 |
| 交易筆數 | 23 | 9 |

樣本外對照 exp_002: Sharpe 0.97→0.63, CAGR 4.4%→3.0%, 回撤 -5.2%→-6.2%, 盈虧比 3.72→2.99, 交易 12→9.

### 結論

**假設被推翻 (第二次失敗, 更嚴重).** 移動止損沒拉高報酬反而大跌, exp_002 仍 best-so-far. **關鍵: 與 exp_003 同型失敗 (勝率升、CAGR 降)** — 兩個獨立機制 (放慢慢線 vs 移動止損) 都想「讓贏家跑」卻同型失敗, 構成模式: **BTC 日線上「抱久一點」不會提高報酬反而降低.** 樣本內外同向變差 (in-sample Sharpe 1.54→1.14), 排除運氣. 原因: BTC 趨勢結束傾向急轉, EMA 交叉 (反應轉弱) 勝過移動止損 (等固定回吐). **三個實驗共同證明: 報酬瓶頸不在出場端.** 工程與實驗分開看: 移動止損功能實作正確且測試完整, 是可複用資產; 失敗的是「它能拉高 BTC 報酬」的假設.

### 下一步實驗

**停止在出場端調整** (exp_002/003/004 已判死). 推論: EMA+ADX 趨勢跟蹤在 BTC 日線的 Sharpe 天花板約 ~1.0, exp_002 (0.97) 已逼近. 要過 1.0 需**換 edge** 而非微調同套: 多時間框架確認 (Round 3)、換進場信號、或 regime 過濾, 各需重新提假設. 或接受 exp_002 為 BTC 階段性最佳, **凍結後**跑 ETH/SPY/QQQ 跨市場驗證 (見方法論註記).
